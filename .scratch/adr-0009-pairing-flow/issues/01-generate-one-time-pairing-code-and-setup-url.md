# Generate one-time pairing code and setup URL

Status: production-ready
## Parent

ADR-0009 — `docs/adr/ADR-0009-pairing-flow.md`

## What to build

Implement `mery pair` so the helper generates a short-lived one-time pairing code and prints a clear setup URL and user instructions for Zam Reader Options.

## Acceptance criteria

- [x] Pairing codes use cryptographically secure randomness and are 6-8 alphanumeric characters. Unit tests pin uppercase alphanumeric 6-8 character codes generated by `secrets.choice()` over the pairing alphabet.
- [x] Codes expire after 10 minutes and are not logged in diagnostics. `PairingChallenge.safe_metadata()` exposes only expiry metadata and tests assert pairing code/setup URL are omitted.
- [x] CLI output includes code, setup URL, expiry, and Zam Reader Options instructions. CLI tests assert `Pairing code:`, localhost setup URL, expiry, and Zam Reader Options copy are printed.
- [x] Tests cover code creation, expiry metadata, and no code leakage in logs. Unit coverage pins 6-8 character code creation, 10-minute expiry, safe metadata, and absence of code/setup URL from diagnostic metadata.
